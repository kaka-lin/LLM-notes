# OpenClaw 原生代理人 (Built-in Agents) 設定指南

OpenClaw 核心內建了強大的單一或多代理人 (Multi-agent) 運作能力。不同於外部擴充包，原生的代理人機制強調「輕量化」與「高度自訂性」。

## 1. 代理人目錄架構

> [!TIP]
> OpenClaw 的所有代理人資訊預設存放在使用者的家目錄下（通常位在 `~/.openclaw/`），這使得備份與管理都十分透明。

每個代理人的目錄結構通常如下：

- **`soul.md`**：定義代理人的靈魂、人格與核心任務（系統提示詞）。
- **`history/`**：存放該代理人的對話記憶。
- **`workspace/`**：該代理人的專屬工作目錄，用於讀寫檔案。

## 2. 定義代理人的靈魂：SOUL.md

> [!NOTE]
> `SOUL.md` 是原生代理人的核心，它扮演了 **系統提示詞 (System Prompt)** 的角色，決定了 AI 的專家身分與行為邏輯。

### 2.1 SOUL.md 範本

```markdown
# Agent Identity

你是一個資深的 DevOps 工程師，專精於 Docker 與 CI/CD 流程自動化。

## 行為守則

- 在回答問題前，優先檢查 `workspace/` 底下的環境設定。
- 所有的回覆必須專業、精簡，並提供可執行的 bash 指令。
- 如果需要搜尋網路，請優先調用 `browser` 工具。
```

## 3. 內建代理人互動方式

### 3.1 預設代理人 (main)

當你執行 `./setup.sh` 或首次啟動後，系統會自動建立一個名為 `main` 的預設代理人。

### 3.2 透過 CLI 進行交辦

你可以直接對特定的代理人下達指令：

```bash
# 對名為 main 的助理下令
docker compose run --rm openclaw-cli agent --agent main --message "幫我分析目前的專案結構"
```

## 4. 多代理人協作 (原生模式)

在原生模式下，你可以建立多個獨立資料夾（例如 `agents/researcher`, `agents/writer`），並透過 OpenClaw 的路由功能將複雜任務拆解給不同的「靈魂」處理。

- **架構優勢**：純檔案系統管理，透明度高，且不會引入額外的軟體相依性。
- **管理與擴充**：您可以輕鬆地透過複製整份代理人資料夾來進行遷移或版本控制。
